# A. Jagged Swaps

## 🖼 Problem 10
![Jagged Swaps](/Images/q10.png)

---

**Platform:** Codeforces  
**Topic:** Greedy / Observation  
**Difficulty:** Easy  

---

## 🧠 Idea in One Line
Sorting is possible only if the smallest element is at the start and largest element is at the end.

---

## 🔍 Key Observation
- Operation swaps only local peaks
- Minimum element cannot move left
- Maximum element cannot move right
- So check:
  - first element must be minimum
  - last element must be maximum

---

## 🚀 Approach
- Find minimum and maximum
- Check if first element is minimum
- Check if n is maximum
- If both true → YES else NO

---

## 🪜 Algorithm Steps
1. Read test cases
2. For each test read `n`
3. Read array
4. Find minimum element
5. Find maximum element
6. If `a[0] == min` AND `n == max`
7. Print YES else NO

---

## ⏱ Time Complexity
O(n)

## 📦 Space Complexity
O(n)

---

## ⚠️ Edge Cases
- already sorted array
- reverse sorted array
- n = 3 minimum size
- smallest not at front
- largest not at end

---

## 💻 Code Pattern to Remember
```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    int t;
    cin >> t;

    while(t--){
        int n;
        cin >> n;

        vector<int> a(n);
        for(int i=0; i<n; i++)
            cin >> a[i];

        int mn = *min_element(a.begin(), a.end());
        int mx = *max_element(a.begin(), a.end());

        if(a[0]==mn && mx==n)
            cout << "YES" << endl;
        else
            cout << "NO" << endl;
    }

    return 0;
}
